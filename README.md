A map of a Meshtastic node zoo: site subnets, the open ether around them,
and radio link quality (SNR) between everyone heard in it. The number
of sites and nodes is not hard-coded вЂ” everything comes from data; the
current two subnets (`10.77.77.0/24` and `10.88.88.0/24`) are just one
particular case.

Static site with no build step and no dependencies: HTML + CSS + vanilla JS,
rendered as SVG.

## Live mode: network scan

The collector discovers nodes by itself (scans the subnets for an open
TCP API port), queries them, and assembles the map from their nodeDB вЂ”
who each node hears directly, and at what SNR. It never writes to the nodes.

```sh
python3 collector/scan.py            # single pass в†’ data/live.json
python3 collector/scan.py --loop 300 # rescan every 5 minutes
```

The page re-reads `data/live.json` once a minute and re-renders; if the
file is missing, it shows a hint to run the collector. The last scan time
is shown in the legend; stale data (>15 min) gets flagged.

Settings live in [`collector/config.json`](collector/config.json): the
subnet list (its order = the order of bands on the map), timeouts,
"fragile" subnets (after two failed full handshakes a node there is
queried with a light handshake, without downloading its nodeDB), the
outside-world filter (freshness, max node count), known IPв†”radio-id pairs
for when a node is unreachable, and roaming nodes.

## Data format

`data/live.json` is generated by the collector; the renderer knows nothing
about your network вЂ” it just draws whatever is in the file:

- `zones` вЂ” map bands top to bottom, any number of them: `kind: "subnet"`
  (a site) or `"gap"` (a zone-free area between sites вЂ” nothing is drawn
  for it, outside nodes simply live there), `h` вЂ” band height in px
  (optional);
- `nodes` вЂ” `zone` вЂ” band id, `x`/`y` вЂ” position as fractions of the band,
  `mobile: true` вЂ” roaming node (dashed frame), `hint` вЂ” tooltip;
- `links` вЂ” `lan` (wire within a site) or `rf` (radio leg; `snr` in dB,
  `null` + `note` = not measured, `labelT` вЂ” label position along the
  line, 0..1). A leg is directional: `to` is the node that heard the
  other one; opposite legs of the same pair are drawn side by side.
  There are no special "bridges" вЂ” an inter-site link is just another leg;
- `meta.snrScale` вЂ” the "% of ideal" scale: `floor` dB в†’ 0% (red),
  `ideal` dB в†’ 100% (green); leg color is a continuous gradient over that
  percentage, the label shows the SNR, and the exact % is in the leg's
  tooltip.

Hovering over a node highlights its legs and neighbors.
