# Стек «на чистую воду» v2 — переклассификация (2026-07-20)

Ревизия всех подходов к проверке правды о позициях/связях после замеров на живом
меше (7 дней данных, 66 позиционированных узлов, 171 xlink-звено из трассировок).

## Измеренная база: SNR→расстояние в городе не работает

Калибровка `snr = A + B·log10(d)` на всех доступных подмножествах:

| подмножество                       | n  | r     | наклон B     | σ      | ошибка дистанции |
|------------------------------------|----|-------|--------------|--------|------------------|
| наш двор → GPS-соседи (все)        | 39 | −0.32 | −5.1 дБ/дек  | 8.5 дБ | ×46              |
| город, GPS↔GPS из трасс            | 46 | −0.29 | −4.9         | 8.4    | ×50              |
| город, пары ≥300 м                 | 42 | −0.18 | −3.5         | 8.6    | ×298             |
| только стабильные линки (LOS-канд.)| 25 | −0.41 | −7.0         | 8.2    | ×15              |

Вывод: разброс затенения (~8.5 дБ) на порядок больше информативного наклона
(−5 дБ/декаду вместо теоретических −20: выживают только линки со случайным
просветом — survivor bias выполаживает зависимость). Кольцо «1 км» на деле —
«от 70 м до 15 км». **Метрическая мультилатерация по SNR мертва**; гейт
`r ≤ −0.4` оставлен — если геометрия меша изменится (разнесённые якоря,
войдрайвинг мобильной ноды), она оживёт сама.

Вторая проблема: база своих якорей — **360 метров** (все 4 ноды рядом).
Триангуляция «от себя» слепа по дальности; спасают только городские якоря
(GPS/адресные узлы), полученные через xlink-граф трассировок.

## Классы доверия позиции (`node.posCls`)

| класс | что это                            | источник                          |
|-------|------------------------------------|-----------------------------------|
| **A** | размещено вручную / подтверждено   | config.geo; геокод, сверенный с GPS |
| **B** | заявленный GPS, не опровергнут     | позиция из эфира, posSus нет      |
| **C** | заявленный GPS опровергнут физикой | posSus: прямой приём несовместим с дальностью |
| **D** | адрес из имени (мягкий якорь)      | Nominatim-геокод без сверки       |
| **E** | оценка по графу/сигналу            | est: центроид слышащих + xlink-партнёров, unc честная |
| **F** | позиции нет                        | —                                 |

## Компоненты: живые / спящие / новые

**Живые (без изменений):**
- `posSus` — детектор вранья GPS. Работает, потому что опирается не на точную
  дальность, а на «прямой приём с 300 км физически невозможен».
- `traceNbr` — подтверждение прямого соседства трассировкой (наземная правда
  против переизлучённых копий в nodeDB).
- Адресные якоря (геокодер) — 11 узлов, единственный OSM-вклад, который платит.

**Спящие (за гейтом):**
- Точная мультилатерация SNR-кольцами — выключена калибровочным гейтом,
  включится сама при r ≤ −0.4.

**Новые (эта ревизия):**
- **Исходящие плечи** (`link.snrOut/outTs`): из `snrTowards` трассировок знаем,
  как соседи слышат НАС (пассивно доступно только обратное). Асимметрия
  систематически против нас — до Δ=14 дБ; диагностика антенн/мощности.
- **Графовое размещение** (est v2): центроид не только своих слышащих площадок
  (это «двор», база 360 м), но и городских xlink-партнёров с известными
  позициями. Метрики дальности нет — вес только монотонный по SNR;
  неопределённость = разлёт якорей с полом в медианную длину линка меша.
- **Призраки** (`data.ghosts`): узлы, которых мы вообще не слышим, но которые
  видны в xlink-графе с ≥2 позиционированными партнёрами — наносятся на
  гео-карту пунктиром («слышен в меше, вне нашего слуха»).

**Отвергнуто по данным:**
- Снап к домам OSM — при км-неопределённости это ложная точность.
- Физика распространения по 3D-городу — data-hungry, ненадёжно при 4 нодах.
