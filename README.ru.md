# meshtastic-zoo 📡

[English](README.md) | **Русский**

Карта зоопарка Meshtastic-нод: площадки-подсети, вольный эфир вокруг них
и качество радиолиний (SNR) между всеми, кто в нём слышен. Число площадок
и нод не зашито в код — всё берётся из данных; текущие две подсети
(`10.77.77.0/24` и `10.88.88.0/24`) — просто частный случай.

Статичный сайт без сборки и зависимостей: HTML + CSS + vanilla JS, рендер в SVG.

## Посмотреть

```sh
python3 -m http.server 8814
# → http://localhost:8814
```

## Live-режим: скан сети

Сборщик сам находит ноды (скан подсетей на открытый TCP-порт API),
опрашивает их и собирает карту из их nodeDB — кого нода слышит напрямую,
с каким SNR. Ничего в ноды не пишет.

```sh
python3 collector/scan.py            # один проход → data/live.json
python3 collector/scan.py --loop 300 # пересканировать каждые 5 минут
```

Страница раз в минуту перечитывает `data/live.json` и перерисовывается;
если файла нет — подсказывает запустить сборщик. Время последнего скана
видно в легенде; протухшие данные (>15 мин) помечаются.

Настройки — [`collector/config.json`](collector/config.json): список
подсетей (порядок = порядок полос на карте), таймауты, «хрупкие» подсети
(там после двух неудачных полных хендшейков нода опрашивается лёгким,
без выгрузки nodeDB), фильтр внешнего мира (свежесть; лимит числа нод, 0 — показывать всех),
известные IP↔radio-id на случай недоступности ноды, кочующие ноды.

## Формат данных

`data/live.json` генерирует сборщик; рендерер ничего не знает про твою
сеть — он просто рисует то, что лежит в файле:

- `nodes` — ноды: `x`/`y` — позиция в долях холста (результат силовой
  укладки сборщика), `own: true` — своя нода (синяя карточка),
  `hw` — модель железа (фото + имя в тултипе), `online`/`heard` — статус
  и «когда видели», `mobile: true` — кочующая нода (пунктирная рамка),
  `hint` — тултип;
- `links` — радиоплечи (`type: "rf"`): `snr` в dB (`null` = нет данных —
  серая стрелка), `heard` — когда слышали (epoch), `labelT` — положение
  подписи вдоль линии 0..1. Плечо направленное: `to` — кто услышал;
  встречные плечи одной пары рисуются рядом. Никаких особых «мостов» и
  «LAN-линий» нет — межплощадочный линк это просто ещё одно плечо;
- `meta.snrScale` — шкала «% от идеала»: `floor` дБ → 0% (красный),
  `ideal` дБ → 100% (зелёный); цвет плеча — непрерывный градиент по этому
  проценту, подпись — SNR, точный % — в тултипе плеча.

Карта целиком вписывается в окно — прокрутка не нужна. Зон и полос нет:
раскладка честная — позиции всех нод считает силовая укладка из качества
плеч: чем лучше пара слышит друг друга (зеленее плечо), тем ближе их
карточки. Несвязанные пары активно расталкиваются — их дистанция ничем
не измерена, поэтому они разбегаются по свободному месту и разгружают
схему. Финальную посадку делает страница под реальные пропорции окна:
поворот главной осью вдоль длинной стороны, один масштаб по обеим осям
(пропорции дистанций сохраняются), пересадка при ресайзе окна. Свои
ноды — синие карточки с IP, внешние — чёрные. Между сканами карта не
скачет: прошлые позиции служат затравкой следующей укладки.

Против наложения стрелок: точки входа плеч раскладываются веером по
граням карточек (каждой стрелке — свой порт), встречные плечи одной
пары идут рядом, перекрывшиеся карточки раздвигаются. Плечи внешних нод приглушены и
вспыхивают при наведении. Наведение на ноду подсвечивает её линки и
соседей, **клик — открывает панель подробностей**: фото девайса, ID,
IP, модель, батарея, аптайм, занятость эфира и все плечи с возрастом.

Между своими нодами (найденными по IP) всегда рисуются обе стрелки:
недостающее в скане направление добирается из прошлого `live.json`
(кэш, TTL в `cacheMaxAgeH`), а если данных нет совсем — серое «не изм.».
На карточке виден статус: зелёная точка — нода отвечает по TCP, иначе
бейдж «сколько назад её слышали» (старше 3 ч — оранжевый).

Изображения устройств — официальные рендеры из репозитория
[Meshtastic web-flasher](https://github.com/meshtastic/web-flasher)
(`img/hw/`); незнакомая модель получает картинку-заглушку.
